# SpringBoot テスト対策まとめ（演習回答zipスタイル準拠）

このメモは、演習回答zip（Controller / Form / templates / Repository の書き方）を“そのまま真似できる形”で整理したものです。  
**目的：似た問題が出ても、このメモを見れば実装できる**（分野別・コード解説付き・講義範囲を網羅）。

---

## 0. まず覚える全体像（MVC）

**リクエスト → Controller →（Service/Repository）→ Model → Thymeleaf(View) → レスポンス**

- **Controller**：URL受付、入力を受ける、Modelに詰める、画面を返す
- **Model**：画面に渡すデータの箱（`Model model` → `addAttribute`）
- **View**：`src/main/resources/templates/*.html`（Thymeleaf）

---

## 1. Web基礎（WWW / URL / HTTP）

### WWW
Webページ同士がリンクで繋がって閲覧できる仕組み（ブラウザで見る世界）。

### URL
例：`http://localhost:8080/ex01`
- `http`：プロトコル
- `localhost`：サーバ名（自分のPC）
- `8080`：ポート（SpringBootの標準）
- `/ex01`：パス（ControllerのURL）

### HTTP（超頻出）
- **GET**：取得（画面表示など）
- **POST**：送信（登録・計算・ログインなど）

---

## 2. SpringBoot / Tomcat（最低限）
- SpringBootは **Tomcat内蔵**（別途Tomcatを立てなくても動く）
- Controllerの返す文字列が **テンプレート名** になる（Thymeleaf）

---

## 3. ディレクトリ構成
- Java：`src/main/java/com/example/...`
  - `controller`：受付
  - `form`：入力の器
  - `domain`：業務データ（Entityっぽいクラス）
  - `service`：処理の分離（計算など）
  - `repository`：DBアクセス（JDBC）
  - `constant`：Enum（選択肢など）
- HTML：`src/main/resources/templates/`
- 設定：`src/main/resources/application.yml`
- SQL：`sql/sql.txt`

## 4. SpringMVC（Controllerの書き方：演習zipスタイル）

### 4-1. 画面表示（フォワード）
**最小のController（演習zipの Ex01 形式）**
```java
@Controller
@RequestMapping("/sample")
public class SampleController {

    @RequestMapping("")
    public String index() {
        return "sample"; // templates/sample.html
    }
}
```

**解説**
- `@RequestMapping("/sample")`：URLの共通部分
- `@RequestMapping("")`：`/sample` にアクセスされた時
- `return "sample"`：`templates/sample.html` を表示（フォワード）

**よくあるミス**
- `return "sample.html"` と書かない（演習zipは一部例外があるが、基本は拡張子なし）

---

### 4-2. Modelで値を渡す
```java
@RequestMapping("")
public String index(Model model) {
    model.addAttribute("msg", "Hello");
    return "sample";
}
```

**解説**
- `addAttribute("名前", 値)` で View に渡す
- View側は `${msg}` で取り出す

---

### 4-3. フォワードとリダイレクト
**フォワード（通常のreturn）**
```java
return "result";
```
- URLは変わらない（サーバ内部で画面遷移）

**リダイレクト**
```java
return "redirect:/sample";
```
- URLが変わる（ブラウザが `/sample` に再アクセス）
- 登録・更新後の二重送信対策（PRG）で使う

## 5. リクエストパラメータ受け取り

### 5-1. 直接（引数）で受け取る（単発ならこれ）
HTML
```html
<form th:action="@{/sample/save}" method="post">
  <input type="text" name="name">
  <button>送信</button>
</form>
```

Controller
```java
@RequestMapping("/save")
public String save(String name, Model model) {
    model.addAttribute("name", name);
    return "result";
}
```

**解説**
- `<input name="name">` と `String name` が一致すると自動で入る
- 1〜2項目のときに便利

---

### 5-2. Formクラスで受け取る（複数項目の本命）
Form
```java
public class UserForm {
    private String name;
    private Integer age;

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public Integer getAge() { return age; }
    public void setAge(Integer age) { this.age = age; }
}
```

Controller（演習zipの Ex06 形式）
```java
@Controller
@RequestMapping("/user")
public class UserController {

    @ModelAttribute
    public UserForm setUpForm() {
        return new UserForm();
    }

    @RequestMapping("")
    public String index() {
        return "user";
    }

    @RequestMapping("/save")
    public String save(UserForm form, Model model) {
        model.addAttribute("name", form.getName());
        model.addAttribute("age", form.getAge());
        return "result";
    }
}
```

**解説（重要）**
- `@ModelAttribute`：画面表示前に空のFormを用意（`th:object` で使える）
- Controller引数に `UserForm form` を置くだけで自動バインド

## 6. Thymeleaf（Viewの書き方：講義目次を網羅）

### 6-1. 変数式 `${...}`（Modelの値を取る）
```html
<span th:text="${msg}"></span>
```
- `${msg}` は Controller の `model.addAttribute("msg", ...)` に対応
- `th:text` は **HTMLエスケープされる** → XSS対策（安全）

---

### 6-2. `th:action` と URL式 `@{...}`
```html
<form th:action="@{/save}" method="post">
```
- `@{/save}`：コンテキストパスを意識した安全なURL生成

リンクも同じ：
```html
<a th:href="@{/edit(id=${item.id})}">編集</a>
```

---

### 6-3. Form連携（`th:object` / `th:field`）
HTML（演習zipスタイル）
```html
<form th:action="@{/user/save}" th:object="${userForm}" method="post">
  <input type="text" th:field="*{name}">
  <input type="text" th:field="*{age}">
  <button>送信</button>
</form>
```

**解説**
- `th:object="${userForm}"`：このフォームは `userForm` を使う
- `th:field="*{name}"`：`userForm.name` と自動連携（name属性も自動）

---

### 6-4. 繰り返し `th:each`
```html
<tr th:each="item : ${list}">
  <td th:text="${item.name}"></td>
</tr>
```
- `${list}` は Controller 側の `model.addAttribute("list", list)`
- `item` は1件ずつの変数

---

### 6-5. 条件分岐 `th:if` / `th:unless`
```html
<div th:if="${flag}">OK</div>
<div th:unless="${flag}">NG</div>
```

---

### 6-6. メッセージ式 `#{...}`（国際化）
```html
<span th:text="#{app.title}"></span>
```
- `messages.properties` を使う（講義範囲）
- 演習zipでは多用していないが、**式の種類として覚える**

---

### 6-7. 文字列結合・式
```html
<span th:text="'Hello ' + ${name}"></span>
```

---

### 6-8. 日付/数値フォーマット（講義目次）
（演習zipで必須ではないが、講義範囲として）
```html
<span th:text="${#numbers.formatInteger(price, 3, 'COMMA')}"></span>
<span th:text="${#temporals.format(date, 'yyyy/MM/dd')}"></span>
```

---

### 6-9. テキスト以外の入力（よく使う）
**ラジオ**
```html
<input type="radio" th:field="*{gender}" value="1">男
<input type="radio" th:field="*{gender}" value="2">女
```

**チェックボックス（複数）**
```html
<input type="checkbox" th:field="*{hobbyList}" value="1">音楽
<input type="checkbox" th:field="*{hobbyList}" value="2">映画
```
- Form側は `List<Integer> hobbyList;` のようにする（演習zipのEx17）

**セレクト**
```html
<select th:field="*{departmentId}">
  <option th:each="d : ${departmentList}"
          th:value="${d.id}"
          th:text="${d.name}">
  </option>
</select>
```

## 8. 2重送信対策（PRG） + FlashAttribute（演習zip：Ex16/Ex17/Ex19）

### 8-1. PRGの形
```java
@RequestMapping("/save")
public String save(UserForm form, RedirectAttributes ra) {
    ra.addFlashAttribute("msg", "OK");
    return "redirect:/user/result";
}

@RequestMapping("/result")
public String result() {
    return "result";
}
```

**解説**
- POSTのあとに `redirect:` でGETへ飛ばす → 更新連打で同じPOSTが飛ばない
- `addFlashAttribute`：リダイレクト先の1回だけ使えるデータ

## 9. 入力値チェック（Validation）※テスト範囲

### 9-1. Formにアノテーション（演習zip：Ex19Form / Ex17QuestionnaireForm）
```java
public class UserForm {

    @NotBlank
    private String name;

	@Size(min=1, max=127)

	@Email

	@NotEmpty()
	チェックボックス時に使用

    @Pattern(regexp="^[0-9]+$")
    private String age;

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public String getAge() { return age; }
    public void setAge(String age) { this.age = age; }
}
```

**解説**
- `@NotBlank`：空・空白だけを禁止
- `@Pattern`：正規表現で条件を指定（数字のみ等）
- `Integer` で受けるより、最初は `String` で受けてバリデーションし、OKなら変換が安全

---

### 9-2. Controller側（`@Validated` と `BindingResult` はセット）
```java
@RequestMapping("/save")
public String save(@Validated UserForm form, BindingResult result, Model model) {

    if (result.hasErrors()) {
        return "user"; // 入力画面へ戻す
    }

    return "result";
}
```

**超重要ルール**
- `BindingResult` は **@Validatedの直後**に置く（順番が違うとエラーが拾えない）

---

### 9-3. HTML側（エラー表示）
```html
<input type="text" th:field="*{name}">
<div th:errors="*{name}"></div>
```

### 11-1. Repositoryの基本形（JdbcTemplate + RowMapper）
```java
@Repository
public class EmployeeRepository {

    @Autowired
    private JdbcTemplate jdbcTemplate;

    private RowMapper<Employee> rowMapper = (rs, i) -> {
        Employee e = new Employee();
        e.setId(rs.getInt("id"));
        e.setName(rs.getString("name"));
        return e;
    };

    public List<Employee> findAll() {
        String sql = "SELECT * FROM employees ORDER BY id";
        return jdbcTemplate.query(sql, rowMapper);
    }
}
```

**解説**
- `@Repository`：DBアクセス担当
- `JdbcTemplate`：SQLを実行する道具
- `RowMapper`：DBの1行（ResultSet）→ Javaオブジェクトに変換
- `query`：複数件取得（List）

---

### 11-2. 1件取得（`queryForObject`）
```java
public Employee load(Integer id) {
    String sql = "SELECT * FROM employees WHERE id = ?";
    return jdbcTemplate.queryForObject(sql, rowMapper, id);
}
```
- `queryForObject`：1件

---

### 11-3. 更新（INSERT/UPDATE/DELETE：`update`）
```java
public void deleteById(Integer id) {
    String sql = "DELETE FROM employees WHERE id = ?";
    jdbcTemplate.update(sql, id);
}
```
- `update`：更新系全部（INSERT/UPDATE/DELETE）

---

### 11-4. LIKE検索（演習zip：Ex24DepartmentRepository）
```java
public List<Department> findByName(String name) {
    String sql = "SELECT * FROM departments WHERE name LIKE ?";
    return jdbcTemplate.query(sql, rowMapper, "%" + name + "%");
}
```

---

### 11-5. INSERT時に採番されたIDを取得（講義目次 + 演習zip方向性）
演習zipは `SimpleJdbcInsert` を使う流れが多いです。基本形：
```java
@PostConstruct
public void init() {
    insert = new SimpleJdbcInsert(jdbcTemplate)
            .withTableName("employees")
            .usingGeneratedKeyColumns("id");
}

public Integer insertAndReturnId(Employee emp) {
    SqlParameterSource param = new BeanPropertySqlParameterSource(emp);
    Number key = insert.executeAndReturnKey(param);
    return key.intValue();
}
```

**解説**
- `serial` や `identity` の主キーをINSERT時に自動採番 → その値を取得できる
- 「登録→詳細画面へ」みたいな問題で使う

---

### 11-6. JOIN（演習zip：Ex26EmployeeRepository）
**列名が被るので別名（alias）を付けてRowMapperで読む**
```sql
SELECT e.id e_id, e.name e_name, d.id d_id, d.name d_name
FROM employees e
JOIN departments d ON e.department_id = d.id
```

RowMapper側：
```java
e.setId(rs.getInt("e_id"));
e.setName(rs.getString("e_name"));
e.setDepartmentName(rs.getString("d_name"));
```

【Controller】
```java
package com.example.controller;

import java.util.List;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;

import com.example.domain.Employee;
import com.example.form.UpdateEmployeeForm;
import com.example.service.EmployeeService;

@Controller
@RequestMapping("/employee")
public class EmployeeController {

	@Autowired
	private EmployeeService employeeService;

	@RequestMapping("/showList")
	public String showList(Model model) {
		// 従業員情報一覧の取得
		List<Employee> employeeList = employeeService.showList();
		// モデルに従業員情報を格納
		model.addAttribute("employeeList", employeeList);

		return "employee/list";
	}

	@RequestMapping("/showDetail")
	public String showDetail(String id, Model model) {
		// idをInteger型に変換
		Integer employeeId = Integer.parseInt(id);
		// 従業員情報の取得
		Employee employee = employeeService.showDetail(employeeId);
		// モデルに従業員情報を格納
		model.addAttribute("employee", employee);

		return "employee/detail";
	}

	@RequestMapping("/update")
	public String update(UpdateEmployeeForm updateEmployeeForm, Model model) {
		// idをInteger型に変換
		Integer employeeId = Integer.parseInt(updateEmployeeForm.getId());
		// dependentsCountをInteger型に変換
		Integer dependentsCount = Integer.parseInt(updateEmployeeForm.getDependentsCount());

		// 従業員情報を取得
		Employee employee = employeeService.showDetail(employeeId);
		// 扶養人数を設定
		employee.setDependentsCount(dependentsCount);
		// 従業員情報を更新
		employeeService.update(employee);

		return "redirect:/employee/showList";
	}

}

【Repository】
```java
package com.example.repository;

import java.util.List;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.jdbc.core.BeanPropertyRowMapper;
import org.springframework.jdbc.core.RowMapper;
import org.springframework.jdbc.core.namedparam.BeanPropertySqlParameterSource;
import org.springframework.jdbc.core.namedparam.MapSqlParameterSource;
import org.springframework.jdbc.core.namedparam.NamedParameterJdbcTemplate;
import org.springframework.stereotype.Repository;

import com.example.domain.Employee;

@Repository
public class EmployeeRepository {

	@Autowired
	private NamedParameterJdbcTemplate template;

	private static final RowMapper<Employee> EMPLOYEE_ROW_MAPPER = new BeanPropertyRowMapper<>(Employee.class);

	/**
	 * 従業員情報一覧を取得する.
	 * 
	 * @return 従業員情報一覧
	 */
	public List<Employee> findAll() {
		String sql = "SELECT id, name, image, gender, hire_date, mail_address, zip_code, address, telephone, salary, characteristics, dependents_count FROM employees ORDER BY hire_date DESC";
		List<Employee> employeeList = template.query(sql, EMPLOYEE_ROW_MAPPER);
		return employeeList;
	}

	/**
	 * 従業員情報詳細を取得する.
	 * 
	 * @param id id
	 * @return 従業員情報詳細
	 */
	public Employee load(Integer id) {
		String sql = "SELECT id, name, image, gender, hire_date, mail_address, zip_code, address, telephone, salary, characteristics, dependents_count FROM employees WHERE id = :id";
		MapSqlParameterSource param = new MapSqlParameterSource();
		param.addValue("id", id);
		Employee employee = template.queryForObject(sql, param, EMPLOYEE_ROW_MAPPER);
		return employee;
	}

	/**
	 * 従業員情報を更新する.
	 * 
	 * @param employee 従業員情報
	 */
	public void update(Employee employee) {
		String sql = "UPDATE employees SET name=:name, image=:image, gender=:gender, hire_date=:hireDate, mail_address=:mailAddress, zip_code=:zipCode, address=:address, telephone=:telephone, salary=:salary, characteristics=:characteristics, dependents_count=:dependentsCount WHERE id=:id";
		BeanPropertySqlParameterSource param = new BeanPropertySqlParameterSource(employee);
		template.update(sql, param);
	}

}

# Spring JDBC 基本要素まとめ（EmployeeRepositoryで使うクラス）

このドキュメントは、Spring JDBCでよく使う以下の要素の役割・使い方を初心者向けにまとめたものです。

- List<>
- RowMapper<>
- BeanPropertyRowMapper<>()
- template.query(...)
- MapSqlParameterSource
- template.queryForObject(...)
- BeanPropertySqlParameterSource

---

# 1️⃣ List<>

## 役割
複数データをまとめて持つコレクション

## 使う場面
DBの結果が複数行（一覧）

## 例
```java
List<Employee> employeeList = employeeRepository.findAll();
```

## イメージ
DB
```
1 山田
2 佐藤
3 鈴木
```

Java
```
Employee
Employee
Employee
```

→ まとめて `List<Employee>`

---

# 2️⃣ RowMapper<>

## 役割
DBの1行 → Javaオブジェクトに変換する仕組み

DBは表形式、Javaはオブジェクトなので変換が必要

### これはなに？

データベースの1行を、Javaの「1人分のデータ」に変える人。

### たとえ

英語を日本語に翻訳する人。

データベースの形 → Javaの形 に変える。
---

# 3️⃣ BeanPropertyRowMapper<>()

## 役割
RowMapperの自動版

DB列名とJavaフィールド名を対応して自動マッピング

### これはなに？

自動で翻訳してくれる便利な人。

### なにが便利？

名前が似ていれば勝手に対応してくれる。

## 例
```java
private static final RowMapper<Employee> EMPLOYEE_ROW_MAPPER
    = new BeanPropertyRowMapper<>(Employee.class);
```

## 自動対応例
DB列
```
mail_address
dependents_count
```

Javaフィールド
```
mailAddress
dependentsCount
```

---

# 4️⃣ template.query(sql, EMPLOYEE_ROW_MAPPER)

## 役割
SELECTで複数行取得

## 戻り値
`List<Employee>`

### これはなに？

データベースから「たくさん」データを取ってくる命令。

### 使うとき

一覧を表示するとき。

## 例
```java
String sql = "SELECT * FROM employees";
List<Employee> list = template.query(sql, EMPLOYEE_ROW_MAPPER);
```

## 使う場面
一覧取得

---

# 5️⃣ MapSqlParameterSource

## 役割
SQLパラメータを安全に渡すための箱

SQLインジェクション防止・可読性向上

### これはなに？

データベースから「1つだけ」取ってくる命令。

### 使うとき

詳細画面など、1人だけ欲しいとき。

## 例
```java
MapSqlParameterSource param = new MapSqlParameterSource();
param.addValue("id", id);
```

SQL
```sql
WHERE id = :id
```

---

# 6️⃣ template.queryForObject(sql, param, EMPLOYEE_ROW_MAPPER)

## 役割
SELECTで1行取得

## 戻り値
`Employee`

### これはなに？

Javaのデータを、そのままデータベース用に変えてくれるもの。

### たとえ

持ち物チェックを自動でしてくれる係。

## 例
```java
Employee employee =
    template.queryForObject(sql, param, EMPLOYEE_ROW_MAPPER);
```

## 使う場面
詳細画面・1件取得

---

# 7️⃣ BeanPropertySqlParameterSource

## 役割
Javaオブジェクト → SQLパラメータに自動変換

## 例
```java
BeanPropertySqlParameterSource param =
    new BeanPropertySqlParameterSource(employee);
```

Employeeフィールド
```
name
salary
id
```

SQLパラメータ
```
:name
:salary
:id
```

---

# UPDATE例（全体の流れ）

```java
String sql =
 "UPDATE employees SET name=:name, salary=:salary WHERE id=:id";

BeanPropertySqlParameterSource param =
    new BeanPropertySqlParameterSource(employee);

template.update(sql, param);
```

---

# SELECT一覧の流れ

```java
String sql = "SELECT * FROM employees";

List<Employee> list =
    template.query(sql, EMPLOYEE_ROW_MAPPER);
```

DB複数行 → RowMapper → Employee → List

---

# SELECT1件の流れ

```java
String sql = "SELECT * FROM employees WHERE id=:id";

MapSqlParameterSource param = new MapSqlParameterSource();
param.addValue("id", id);

Employee emp =
    template.queryForObject(sql, param, EMPLOYEE_ROW_MAPPER);
```

---

# 使い分けまとめ

| 目的 | 使うもの |
|------|---------|
一覧取得 | template.query |
1件取得 | template.queryForObject |
少数パラメータ | MapSqlParameterSource |
オブジェクト丸ごと | BeanPropertySqlParameterSource |
DB→Java変換 | RowMapper |
自動変換 | BeanPropertyRowMapper |
複数結果 | List<> |

---

# 全体イメージ

```
DB → RowMapper → Employee → List

Employee → BeanPropertySqlParameterSource → SQL
```
